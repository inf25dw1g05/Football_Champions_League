# C2 : Resources

Nesta secção apresentamos os recursos disponibilizados no nosso projeto.  
Todos os endpoints foram documentados no ficheiro `openapi.yaml` e implementados através de controladores e serviços específicos.

A API inclui recursos relacionados com:

---

## **1. Equipas**
Permite:
- Criar uma equipa
- Consultar uma equipa
- Listar jogos de uma equipa
- Listar jogadores de uma equipa
- Listar todas as equipas
- Atualizar uma equipa
- Remover uma equipa

Inclui também a relação entre jogadores, treinadores e grupos.

---

## **2. Jogadores**
Permite:
- Criar um jogador
- Remover um jogador
- Listar eventos associados a um jogador específico
- Consultar um jogador
- Listar todos os jogadores (com opções de filtragem, como posição ou nacionalidade)
- Atualizar um jogador

---

## **3. Jogos**
Permite:
- Criar um jogo
- Apagar um jogo
- Consultar um jogo (equipas, estádio, data, resultado)
- Listar jogos com filtros por equipa, data, grupo ou estádio
- Atualizar as informações de um jogo específico

---

## **4. Eventos de Jogo**
Permite:
- Criar um evento
- Apagar um evento
- Listar todos os eventos de um jogo
- Atualizar um evento (ex.: anular um golo)

---

## **5. Estatísticas**
Permite:
- Listar estatísticas disciplinares (jogadores com mais sanções)
- Consultar a classificação por grupo
- Listar os jogadores com mais assistências
- Listar os jogadores com mais golos

---

## **6. Estrutura Docker**
A API é suportada por um sistema multicontainer composto por:
- `api-service` (backend)
- `db-service` (base de dados)
- volumes persistentes para armazenamento

Toda a interação com os recursos é automaticamente disponibilizada após executar:
`docker compose up`
